## Brand Overview

### Brand Essence

**SuperInstance AI** represents a new paradigm in artificial intelligence: privacy-first, local-first, and community-driven. Our brand reflects our core values of transparency, security, and user empowerment.

### Core Values

- **Privacy First**: Your data belongs to you
- **Local by Default**: Process locally, escalate intelligently
- **Transparent**: Open source, auditable, explainable
- **Collaborative**: Tripartite consensus, community-driven
- **Professional**: Technical excellence without pretension

### Brand Personality

- **Expert**: Deep technical knowledge
- **Approachable**: Complex concepts made accessible
- **Trustworthy**: Privacy and security as foundational pillars
- **Innovative**: Cutting-edge technology with practical application
- **Pragmatic**: Real solutions to real problems

### Typography Best Practices

1. **Max Line Length**: 60-80 characters for optimal readability
2. **Paragraph Spacing**: 1.5x line height between paragraphs
3. **Heading Hierarchy**: Never skip heading levels (H1 ‚Üí H2 ‚Üí H3)
4. **Emphasis**: Use bold sparingly, avoid ALL CAPS
5. **Lists**: Use bullet lists for options, numbered lists for steps

## Logo Usage

### Logo Concept

**Logo Design Specification**: See [assets/logo/logo.md](assets/logo/logo.md) for complete logo design details.

**Brief Summary**:
- **Mark**: Triangular arrangement of three interconnected nodes (tripartite concept)
- **Wordmark**: "SuperInstance" in bold, "AI" in lighter weight
- **Symbolism**: Three agents working together, connected intelligence

### Logo Variations

1. **Primary Logo**: Full color (horizontal)
2. **Icon Only**: Mark only (applications, favicon)
3. **Stacked**: Vertical arrangement (social media, cards)
4. **Monochrome**: Single color (single-color applications)

### Logo Clear Space

**Minimum Clear Space**: Height of the "S" in SuperInstance

**Never**:
- Place logo too close to other elements
- Crowd the logo with text or graphics
- Stretch or distort the logo
- Use low-resolution versions

### Minimum Size

**Print**: 1.5 inches (38mm) wide
**Digital**: 120px wide
**Favicon**: 32x32px (icon only)

### Logo Backgrounds

**Approved Backgrounds**:
- White
- Deep Space Blue (#0A1628)
- Transparent (with adequate contrast)

**Avoid**:
- Busy patterns
- Colors with insufficient contrast
- Photos (unless adequate clear space)

## Icon & Emoji Guidelines

### Icon Philosophy

Icons should be simple, clear, and meaningful. Use icons to enhance comprehension, not as decoration.

### Icon Usage

**DO**:
- Use icons consistently (same icon = same meaning)
- Ensure icons are understandable without labels
- Maintain visual consistency (same style throughout)
- Test icon recognition with users

**DON'T**:
- Overuse icons (not every element needs an icon)
- Mix icon styles (outline vs. filled)
- Rotate or flip icons (unless semantically appropriate)
- Use icons as the only means of communication

### Strategic Emoji Usage

**Philosophy**: Emojis should add personality, not overwhelm content.

## Voice & Tone

### Voice

SuperInstance's voice is **professional, technical, and approachable**.

**Characteristics**:
- **Expert but not elitist**: We know our stuff, but we explain clearly
- **Precise but not pedantic**: Technical accuracy without unnecessary jargon
- **Confident but not arrogant**: We're good at what we do, but we're still learning
- **Human but not casual**: Professional warmth without being overly informal

### Tone

**Tone varies by context**:

| Context | Tone | Example |
|---------|------|---------|
| Documentation | Instructional, clear | "To install SuperInstance, run..." |
| Error Messages | Helpful, non-blaming | "Couldn't connect to server. Check your network connection." |
| Marketing Copy | Inspiring, confident | "Experience AI that respects your privacy." |
| Technical Specs | Precise, concise | "Supports CUDA 11.0+ with 4GB+ VRAM" |
| Community | Welcoming, appreciative | "Thanks for your contribution!" |

### Writing Guidelines

**DO**:
- Use active voice: "Install the package" not "The package should be installed"
- Be direct: "Run this command" not "You might want to consider running"
- Use second person: "Your data" not "User data"
- Break up complex concepts: Use examples, analogies, diagrams
- Include why, not just what: "Redact sensitive data to protect privacy"

**DON'T**:
- Use jargon unnecessarily: "The tripartite council reaches consensus" (explain first)
- Be wordy: "In order to" ‚Üí "To"
- Use weasel words: "basically", "actually", "just"
- Overpromise: Be honest about limitations
- Use exclamation marks excessively: One per paragraph max

### Accessibility

**Inclusive Language**:
- Use person-first language: "people with disabilities" not "the disabled"
- Avoid gendered pronouns: Use "they" or rephrase
- Avoid idioms: Non-native speakers may not understand
- Provide context: Don't assume prior knowledge

**Clear Communication**:
- Flesch Reading Ease: Target 60-70 (standard/conversational)
- Sentence Length: Average 15-20 words
- Paragraph Length: Max 3-5 sentences
- Use examples for complex concepts

## Visual Hierarchy

### Hierarchy Principles

1. **Size Matters**: Larger = more important
2. **Position Matters**: Top-left = most important (in LTR languages)
3. **Color Matters**: Brighter, more saturated = more important
4. **Contrast Matters**: Higher contrast = more importance
5. **White Space Matters**: More space around = more important

### Header Hierarchy

**Page Header** (H1):
```
SuperInstance AI
Privacy-first, local-first AI with tripartite consensus
```

**Section Header** (H2):
```
## üöÄ Quick Start
```

**Subsection Header** (H3):
```
### Prerequisites
```

### Content Hierarchy

**Priority Order**:
1. Critical actions (CTAs)
2. Key information (what/why)
3. Supporting details (how)
4. Supplementary info (examples, tips)
5. Metadata (dates, versions)

## Dos and Don'ts

### General

**DO**:
- Follow these guidelines consistently
- Test for accessibility (color contrast, screen readers)
- Use high-quality images and graphics
- Maintain clear visual hierarchy
- Keep layouts clean and uncluttered

**DON'T**:
- Stretch or distort logos or icons
- Use colors outside the approved palette
- Crowd elements together
- Use low-resolution images
- Skip heading levels

### Documentation

**DO**:
- Start with a clear title and purpose
- Use code blocks for commands
- Include examples for complex concepts
- Add diagrams for architecture
- Link to related resources

**DON'T****:
- Assume prior knowledge
- Use unclear section titles
- Bury important information
- Overuse formatting (bold, italic, code)
- Include unnecessary details

### Code Examples

**DO**:
- Show complete, runnable examples
- Include expected output
- Comment complex sections
- Follow language conventions
- Test all examples

**DON'T**:
- Use placeholder code that doesn't work
- Omit critical setup steps
- Use confusing variable names
- Overcomplicate examples
- Forget error handling

### Social Media

**DO**:
- Use brand colors in graphics
- Include clear, actionable CTAs
- Use relevant hashtags (#PrivacyFirst #AI #LocalFirst)
- Tag contributors (with permission)
- Share behind-the-scenes content

**DON'T****:
- Post low-quality images
- Use excessive hashtags (max 3-5)
- Overuse emojis (max 1-2 per post)
- Post without proofreading
- Ignore engagement

### Image Dimensions

| Asset Type | Dimensions | Format | Notes |
|------------|-----------|--------|-------|
| Logo (horizontal) | 1200x300px | SVG, PNG | Transparent background |
| Logo (icon only) | 512x512px | SVG, PNG | Square, transparent |
| Social preview | 1280x640px | PNG | For GitHub, Twitter |
| Favicon | 32x32px, 16x16px | ICO, PNG | For browser tabs |
| App icon | 1024x1024px | PNG | For app stores |
| Documentation image | Min 1200px wide | PNG, SVG | Clear, readable |

### File Formats

**Vector** (SVG):
- Scalable without quality loss
- Use for logos, icons, diagrams
- Can be animated

**Raster** (PNG):
- Use for photos, screenshots
- Transparent background supported
- Lossless compression

**JPEG:
- Use for photos only
- Smaller file size than PNG
- No transparency

### File Size Limits

- **Logo (SVG)**: < 50KB
- **Logo (PNG)**: < 200KB
- **Social preview**: < 500KB
- **Documentation images**: < 300KB each
