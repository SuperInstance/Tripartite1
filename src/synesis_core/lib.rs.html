1//! # SuperInstance Core - Agent Orchestration Engine
2//!
3//! This crate implements the heart of the SuperInstance AI system: the **tripartite council**
4//! that combines three specialized AI agents to process queries through consensus.
5//!
6//! ## Architecture Overview
7//!
8//! The SuperInstance system uses three specialized agents, each with a unique perspective:
9//!
10//! - **Pathos** (Intent Agent): Understands what the user truly wants, extracting intent,
11//!   detecting user expertise level, and framing the query for other agents.
12//! - **Logos** (Logic Agent): Performs logical reasoning, retrieves relevant knowledge via RAG,
13//!   and synthesizes comprehensive solutions.
14//! - **Ethos** (Truth Agent): Verifies responses for safety, accuracy, feasibility, and quality.
15//!   Has veto power over dangerous outputs.
16//!
17//! ## Consensus Mechanism
18//!
19//! The three agents work together through a multi-round consensus process:
20//!
21//! 1. **Round 1**: Pathos extracts intent → Logos generates solution → Ethos verifies
22//! 2. **Evaluation**: Weighted voting determines if consensus is reached (default threshold: 0.85)
23//! 3. **Revision**: If consensus isn't reached, feedback is provided and the process repeats
24//! 4. **Outcome**: Returns when consensus is reached, vetoed, or max rounds exceeded
25//!
26//! ## Usage Example
27//!
28//! ```rust,no_run
29//! use synesis_core::{Council, CouncilConfig};
30//!
31//! #[tokio::main]
32//! async fn main() -> Result<(), Box<dyn std::error::Error>> {
33//!     // Create council with default configuration
34//!     let mut council = Council::new(CouncilConfig::default());
35//!
36//!     // Initialize all agents (load models)
37//!     council.initialize().await?;
38//!
39//!     // Process a query
40//!     let manifest = synesis_core::A2AManifest::new(
41//!         "Explain how vector databases work".to_string()
42//!     );
43//!     let response = council.process(manifest).await?;
44//!
45//!     println!("Response: {}", response.content);
46//!     println!("Confidence: {:.2}", response.confidence);
47//!
48//!     Ok(())
49//! }
50//! ```
51//!
52//! ## Module Structure
53//!
54//! - [`agents`]: Agent trait and implementations (Pathos, Logos, Ethos)
55//! - [`consensus`]: Consensus engine and voting mechanism
56//! - [`council`]: High-level council orchestration
57//! - [`manifest`]: A2A manifest for query tracking
58//! - [`routing`]: Local vs. cloud decision routing
59//!
60//! ## Privacy Integration
61//!
62//! The consensus engine integrates with the privacy crate to automatically redact sensitive
63//! information before processing and re-inflate it in responses. This ensures that cloud
64//! processing never sees raw PII or credentials.
